Setup inicial — Nginx wildcard

Nginx

Configurado uma única vez. Cobre qualquer subdomínio presente e futuro sem alterar este arquivo.

bash — criar o arquivo de configuração
nano /etc/nginx/sites-available/bellex
/etc/nginx/sites-available/bellex
# Redireciona HTTP → HTTPS
server {
    listen 80;
    server_name bellex.beauty *.bellex.beauty;
    return 301 https://$host$request_uri;
}

# HTTPS — wildcard, serve todos os subdomínios
server {
    listen 443 ssl;
    server_name bellex.beauty *.bellex.beauty;

    ssl_certificate     /etc/letsencrypt/live/bellex.beauty/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/bellex.beauty/privkey.pem;
    include             /etc/letsencrypt/options-ssl-nginx.conf;

    root  /var/www/bellex;
    index index.html;

    # SPA — todas as rotas caem no index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Proxy da API de IA/WhatsApp
    location /api/ {
        proxy_pass         http://localhost:3001/;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
bash — ativar e testar
ln -s /etc/nginx/sites-available/bellex /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
Setup inicial — DNS wildcard

DNS

No painel de DNS do registrador do domínio bellex.beauty, criar dois registros:

registros DNS
@   A   76.13.225.174   (TTL 300)
*   A   76.13.225.174   (TTL 300)

O registro * cobre automaticamente studio13.bellex.beauty, espacovip.bellex.beauty e qualquer futuro subdomínio — sem tocar no DNS de novo.

Setup inicial — SSL wildcard

Certificado SSL

Um único certificado wildcard cobre todos os subdomínios. Requer DNS challenge (não HTTP challenge).

bash — emitir certificado wildcard
certbot certonly --manual --preferred-challenges dns \
  -d bellex.beauty -d "*.bellex.beauty"
⚠

O certbot vai pedir para adicionar um registro TXT _acme-challenge.bellex.beauty no DNS. Adicionar o registro, aguardar propagar (1–2 min) e só então pressionar Enter para continuar.

Após emitido, o Nginx já usa o certificado via os caminhos configurados acima. Renovação automática via certbot renew (cron do sistema).

Setup inicial — EvoAPI

EvoAPI — servidor (configurar 1x no Workspace)

A URL e a API key do servidor EvoAPI são configuradas uma única vez pelo admin no painel do Workspace.

Caminho: Workspace → Integrações → WhatsApp / EvoAPI → informar URL do servidor e API key.

Essas credenciais ficam salvas no banco e são herdadas por todas as clínicas. A partir daí, cada clínica cria e gerencia sua própria instância diretamente pelo painel — sem nenhum acesso manual ao painel do EvoAPI.

WhatsApp

ℹ

Zero configuração manual no painel EvoAPI. Todo o processo acontece dentro do painel da clínica, usando as credenciais do servidor EvoAPI configuradas no Workspace.

1

Acessar configurações de WhatsApp

No painel da clínica → Configurações → WhatsApp → "Criar instância"

Criação automática da instância

Novo cliente — Passo 5

Agente de IA

A API usa o mesmo código para todas as clínicas — o comportamento do agente é definido por dados no banco, não por código.

⚠

Este é o único passo ainda manual. A UI de configuração do agente no painel Admin ainda não está completa. Por enquanto, executar o SQL abaixo diretamente no Supabase.

SQL — Supabase (manual por enquanto)
-- prompt do agente
INSERT INTO agent_configs (clinica_id, system_prompt, ativo)
VALUES (
  '<clinic_id>',
  'Você é a assistente virtual do Studio 13, clínica de estética da Patricia.
Seu objetivo é qualificar leads, responder dúvidas e auxiliar no agendamento.
Seja simpática, objetiva e profissional.',
  true
);

-- pipeline de atendimento (colunas do CRM + critérios da IA)
-- IMPORTANTE: stage sem agent_id = IA para de responder automaticamente
INSERT INTO pipeline_stages (clinica_id, nome, criterio_movimentacao, agent_id, ordem)
VALUES
  ('<clinic_id>', 'Novo Lead',        'primeiro contato recebido',         '<agent_id>', 1),
  ('<clinic_id>', 'Qualificado',      'paciente demonstrou interesse real',  '<agent_id>', 2),
  ('<clinic_id>', 'Aguarda Humano',   null,                                  null,          3);
--                                                                         ↑ sem agent_id → IA silenciosa
⚠

A IA nunca escala para humano por conta própria. Ela apenas move o lead entre colunas. Quando o lead entra em uma coluna sem agent_id, a IA para de responder. O botão "Pausar IA" no painel é para uso humano — não é uma ação da IA.

Extra — Domínio customizado

Domínio customizado

Para clientes que querem usar um domínio próprio (ex: app.studio13.pt) em vez do subdomínio padrão.

1

DNS (lado da cliente)

No painel de DNS do domínio da cliente, adicionar:

app   CNAME   studio13.bellex.beauty   (TTL 300)
2

Certificado SSL (VPS)

Esta é a única etapa que toca a VPS. O certbot adiciona automaticamente o server block necessário.

certbot --nginx -d app.studio13.pt
3

Registrar no banco

UPDATE workspace_clinics
SET custom_domain = 'app.studio13.pt'
WHERE subdomain = 'studio13';

O frontend detecta o domínio customizado e carrega o tenant correto via CustomDomainGate.
